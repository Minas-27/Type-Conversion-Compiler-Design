# Type Conversion in Compiler Design
### Overview
This repository contains the technical implementation and presentation materials for the study of **Type Conversion** (Implicit Coercion and Explicit Casting). This is a core component of the Semantic Analysis phase in compiler construction.

### Contents
* `type_conversion.cpp`: C++ implementation demonstrating conversion logic.
* `Presentation.pdf`: Slide deck for the technical presentation.
* `Flowchart.png`: Visual representation of the compiler's semantic checking process.

### Key Concepts
1. **Implicit Conversion (Coercion):** Automatic type promotion by the compiler to prevent data loss (e.g., Int to Float).
2. **Explicit Conversion (Casting):** Manual type conversion forced by the programmer, often resulting in data truncation.

### How to Run the Code
1. Clone the repo: `git clone https://github.com/Minas-27/Type-Conversion-Compiler-Design`
2. Compile: `g++ type_conversion.cpp -o conversion`
3. Run: `./conversion`
